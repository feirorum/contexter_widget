People & Topics

Quick glance at people in your vault and the main topics found in their connected snippets.

How this page was created — click to expand

User request (brief): "Take all persons in data-md/people, make a ppl.html showing each person with the main topics found in their connected snippets (use wikilinks in the last heading). Make the page modern, bright and inviting."

Planning (chat): I created a small todo plan, then collected the person files and snippet files to determine the mapping from people → topics. The core steps were:

  • Read all contact/person files under data-md/people.
  • List and read snippet files under data-md/snippets.
  • For each snippet, extract wikilinks found in the last heading/"Related" or references section and use those as topics.
  • Map snippets to people when the snippet references a person (via frontmatter people or wikilink in the content).
  • Generate a compact HTML page with one card per person showing role, a short blurb, and a few topic tags. Use fallbacks to person metadata when snippets lacked topics.

Execution (what was done): I parsed the repository files and created context-tool/data-md/people/ppl.html. Concretely:

  • Read person markdowns (e.g., emma-rodriguez.md, sarah-mitchell.md, etc.).
  • Fetched snippet files and inspected their frontmatter and content to find wikilinks and the "Related"/references section.
  • Extracted topics from snippet wikilinks where present; when snippets had no topics, used the person file's related links/tags as a fallback to keep cards informative.
  • Built a responsive card-based layout with colorful avatars, short blurbs, and topic chips. The page is static and located at the path above.

Files changed: Added data-md/people/ppl.html (static HTML view).

Notes & next steps: I can (a) make a small Python generator script to regenerate this page automatically, (b) switch to strict-only-snippet-topics (no fallbacks), or (c) add interactivity (tag filtering, counts, links to snippets). Tell me which and I will implement it.
